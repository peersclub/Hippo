The model — parasite on a real host

Assetworks Exchange (this host, the light UI at :4000) is a self-contained venue with its own signed trade API, fill engine, balances, and positions, streaming live market data from Binance public WebSockets. Hippo embeds as a parasite via the single /loader.js tag and places orders through the execution seam. Because the host speaks the same signed wire shape as the KoinBX pilot, the parasite drives it through production code paths — HMAC signing, the poll reconciler, capability discovery, and both confirm surfaces — not a sim timer.

The point: an order the conversational parasite places lands in the same backend the human order ticket uses, so it appears in the host's native blotter and moves the same balances — exactly as a real third-party integration would.

Two order paths, one book

PathHow to placeAuth
HUMANThe order ticket on the right of the exchange UI (spot/perp, buy/sell, market/limit).First-party /ui/orders — no signing.
PARASITETalk to Hippo: e.g. “buy 0.01 BTC” → research/ticket → confirm.HMAC-signed /api/v1/trade/orders via the seam.

Both write to the same trader-1 book, so whatever you place either way shows in the Open orders / Positions / Balances blotter (live over SSE) and in Hippo's positions card.

Confirm surfaces — the admin switch Open Decision #6

Open the ⚙ admin drawer (top-right of the exchange) and flip Confirm surface. The parasite reads it live from /admin/config at confirm time — no restart.

SurfaceBehaviour
apiHippo confirms in its own panel and places directly with the scoped key. The order shows up in the host blotter after placement.
js_callbackHippo hands off; the host renders its own confirm modal; nothing is placed until you approve there. Decline → parasite lifecycle shows DECLINED.

Walkthroughs

A · Parasite spot buy, api surface

  1. Admin drawer → Confirm surface = api.
  2. Open Hippo, say “buy 0.01 BTC”. Review the ticket, tap confirm.
  3. Watch the exchange Open orders tab.the order appears (ACTIVE), then flips to FILLED within a few seconds
  4. Check Balances and Hippo's positions card.USDT decreased, BTC increased — same numbers on both sides

B · Parasite order, js_callback surface

  1. Admin drawer → Confirm surface = js_callback.
  2. Ask Hippo to buy again and confirm in Hippo's panel.a native Assetworks confirm modal pops on THIS host, not in Hippo
  3. Click Confirm & place.order places, fills, and Hippo's lifecycle card resolves to FILLED
  4. Repeat and click Decline instead.nothing is placed; Hippo shows DECLINED ON THE VENUE

C · Perps (human ticket)

  1. Ticket → Perp, Buy/Long, leverage 10x, size 0.1, place.
  2. Open the Positions tab.a long with entry, 10x, and a liquidation price below entry
  3. Let the live price move, then place a reduce-only close.position clears; realized PnL lands in the USDT balance
