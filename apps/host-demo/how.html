2 · Client features — test on localhost:4000/?gw=real

Live AI research briefs (OpenRouter · Claude Haiku 4.5)

  1. Open the Hippo bubble (bottom-right) and ask What's driving BTC price today? tokens stream in live; the finished card shows real price, 12h change, funding stats, a sparkline, and the model tag anthropic/claude-haiku-4.5 in the top-right of the card
  2. Ask a concept question: What is a funding rate? a CONCEPT card — explanation prose, no market stats fabricated
  3. Ask the same question again. a CACHED BRIEF badge — answers are cached with volatility-scaled TTL

Numbers are retrieval, never generation

Every stat, sparkline and as-of timestamp comes straight from the live market-data feed (CCXT/Binance) — the model only writes prose. A hallucinated price cannot reach a stat cell.

No-advice guardrail

  1. Ask Should I buy BTC right now? a decline card — "◇ NO ADVICE — BY DESIGN" — that pivots to live facts instead of a recommendation. This is enforced on the model's output, not just the prompt

Order preparation + venue handoff (sim venue)

  1. Type buy 0.05 BTC an order ticket — side, size, live price rows — with "Review & confirm" CTA. Hippo prepares; the exchange confirms. Hippo never executes
  2. Confirm the ticket. lifecycle frames stream in: awaiting confirm → filled, with venue order id

Portfolio

  1. Ask My positions & P&L a positions table frame

Card interactions

  1. On any brief: ↻ REFRESH re-runs with fresh data · ⧉ COPY puts clipboard text · ↗ SHARE opens the share overlay · 👍/👎 feedback (👎 asks one follow-up reason) all work without leaving the card; feedback survives minimize/reopen

Multi-language

  1. Ask in Hindi or Hinglish: BTC ka funding rate kya hai? language detected and the answer follows it

Never-500 degraded mode

Kill the intelligence service (pkill -f uvicorn) and ask a research question — you still get a truthful market-data-only brief plus a "high market load" banner. Orders and prices keep working. Restart pnpm dev to restore.

PlansPilot (3 MAU) / Growth (1000) / Enterprise (unlimited). Create, edit, assign to partners. Delete is blocked while a partner uses the plan.
UsersEnd-users per partner, first/last seen. Block one → that user can no longer mint sessions (401).
MemoryView/edit/clear a user's opt-in persona (experience level etc.). Data is owned by the memory service; admin proxies with an internal token.
OperatorsOwner-gated: add/remove panel operators (owner vs operator roles).
AuditEvery mutating admin action logged — who, what, when. Check it after doing any of the above.

MAU quota enforcement (429)

Plans cap distinct monthly users. When a partner is at quota, a new user's session mint returns 429 — returning users are never cut off mid-month. Bitrupt (3/3) demonstrates this live.

Self-serve sandbox provisioning (API)

curl -X POST http://localhost:8794/v1/provision/sandbox \
  -H "Content-Type: application/json" \
  -d '{"venueName":"My Exchange","email":"you@example.com","locales":["en"]}'

Returns a sandbox partner + a one-time claim URL for the JWT secret (15-min TTL, fetchable exactly once, never appears in any list). Rate-limited to 3/IP/hour. Activation to production stays operator-gated in the panel.

Known gaps (backend works, no UI page yet): a global Sessions list page and a Provisioning approval page in the admin panel — both are reachable via API today (partner-detail has per-partner session kill). Streaming deltas don't show the model tag until the final card lands.
