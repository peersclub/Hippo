HIPPO DESIGN LANGUAGE

Dark Glass Instrument.

One amber accent on deep glass surfaces, data set in mono, answers stamped in time. Dark is the locked hero; light is a secondary lean — a pure token swap, never a redesign. Try the toggle ☀ above: this page, and the Ask Hippo pill, re-skin from the same tokens.

THE RULES THAT DON'T BEND

Six principles.

01

Amber is the only accent

#F0B94A is the single brand signal — CTAs, focus, the mark. If something else wants to be amber, it's wrong.

02

Up and down are semantics

Green means gains/buy, red means losses/sell — never decoration. A colour that carries meaning can't be spent on style.

03

Dark is the locked hero

The product ships dark-first. Light is a supported lean, produced by swapping tokens only — zero layout or rule changes.

04

Answers are facts about a moment

Every brief carries a timestamp and sources. Data is set in IBM Plex Mono so numbers read as instrument, not prose.

05

Solid cards, no blur in scroll

Card backgrounds inside the scroll container are solid; backdrop-filter is reserved for full-surface overlays (iOS/WebKit rule).

06

The client only draws

Cards and capabilities are server-driven. The SDK renders what it's sent — unknown frames get a fallback card, never a crash.

VOICE

Confident. Factual. Never a shill.

Explains, never advises. No signals, no predictions, no "what would you do?" Declines are delivered with conviction — no apology — then pivot to facts that actually help.
Speak the trader's language. English in → English out. हिन्दी in → हिन्दी out. Hinglish in → Hinglish out. RTL scripts get a mirrored chrome.
Answers are facts about a moment. Timestamps and sources on every brief; a cached answer says so, and keeps its original "as of".
Money is never computed client-side. All amounts arrive as pre-formatted strings — Indian digit grouping where the venue trades in ₹.
Failure is honest. Degraded mode says "fresh research may take longer" and keeps orders, prices and saved briefs live. Errors keep the user's words: "Your question is saved — nothing you wrote is lost."
THEME SYSTEM

Theme-aware by token swap.

Every surface resolves through --hippo-* custom properties. Light mode redeclares token values only — no layout, no new rules — so the dark hero stays pixel-locked and a theme can never fork the design. The choice is a manual, persisted toggle (like the exchange apps we embed in), not a media query.
