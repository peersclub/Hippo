HIPPO SDK · INTEGRATION

One tag. Every platform.

The Hippo SDK is a two-stage thin client: a zero-dependency loader that mounts the Ask Hippo pill in a closed Shadow DOM, and a lazily-loaded panel that only ever draws what the server sends. Because the whole product is server-driven cards over HTTPS + SSE, "integrating Hippo" means the same thing everywhere — deliver one script tag, and on native, wrap it in a WebView shell we ship for you.

THE HOST-SAFETY CONTRACT

A parasite that never harms the host.

Closed Shadow DOM

Your CSS can't leak in; ours can't leak out. The SDK creates one <hippo-root> element with its own fixed stacking context — your layout never shifts.

Fails invisible

No globals, no console noise. A loader failure disappears silently rather than erroring the host page. Mounting waits for your load event.

Size-gated in CI

The loader ships under a hard 5KB-gzip gate enforced on every PR (currently ~1.2KB). The 45KB panel loads only on first interaction.

Draws, never computes

Money arrives as pre-formatted display strings. Unknown card types render a fallback card — an old SDK never crashes on a new server.

Additive protocol

Card protocol v1 only ever adds fields. Every improvement ships server-side the moment we ship it — zero upgrade work in your app, ever.

Reconnect-proof

SSE with a sequence-numbered frame journal: after a disconnect the SDK resumes from Last-Event-ID and replays what it missed — order fills survive tunnels.

PLATFORM GUIDES

Where it runs.

One integration model, four delivery mechanisms: a script tag on the web, a WebView shell on native. The panel's posture matrix adapts by viewport — dock / overlay / max on wide screens, sheet / full under 640px — so the same bundle is the desktop widget and the mobile surface.

Web — any site, any stack SCRIPT TAG

Plain HTML, WordPress, Shopify, Webflow, a Google Tag Manager custom tag — anywhere you can paste one script tag, Hippo runs. Server-rendered frameworks (Rails, Django, Laravel, PHP) drop it in the base template.

React / Next.js NPM-STYLE

// AskHippo.tsx — works in Next.js App Router ('use client') and any React app
import { useEffect } from 'react'

export function AskHippo() {
  useEffect(() => {
    const s = document.createElement('script')
    s.src = 'https://cdn.hippo.app/loader.js'
    s.async = true
    s.dataset.hippoKey = 'pk_yourvenue'
    document.body.appendChild(s)
    return () => { s.remove(); document.querySelector('hippo-root')?.remove() }
  }, [])
  return null
}

// open Hippo from your own button, without touching our internals:
document.querySelector('hippo-root')?.dispatchEvent(new Event('hippo:open'))

iOS WKWEBVIEW + SHELL

Load the shipped shell — cdn.hippo.app/embed/mobile.html — in a WKWebView. It auto-opens the panel (no floating pill inside an app), respects safe areas, and bridges events through a script-message handler named hippo.

let config = WKWebViewConfiguration()
config.userContentController.add(self, name: "hippo")
let webView = WKWebView(frame: .zero, configuration: config)
webView.load(URLRequest(url: URL(string:
  "https://cdn.hippo.app/embed/mobile.html?key=pk_yourvenue&theme=dark&locale=hi")!))

// events arrive in userContentController(_:didReceive:)
//   {type:"ready"} · {type:"open"} · seam confirm events as they land
// drive it from native:  webView.evaluateJavaScript("HippoShell.setTheme('light')")

NETWORK SURFACE

Three endpoints and a CSP line.

The SDK talks to exactly one origin — your assigned gateway — over HTTPS and SSE. Sessions are minted by your backend with a partner-signed JWT asserting the venue user; anonymous sessions exist for sandboxes.

CALLPURPOSE
POST /v1/sessionPartner-auth handshake → session id + venue config (name, locales, suggested queries)
GET /v1/streamSSE down-channel — seq-numbered card frames, resumable via Last-Event-ID
POST /v1/turnsUplinks: user text, chip taps, ticket actions, feedback, consent, settings
